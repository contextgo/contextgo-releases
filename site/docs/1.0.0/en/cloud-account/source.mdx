## What the cloud account is used for

The cloud account is used for:

- account sign-in
- desktop device registration
- device discovery from browser or mobile clients
- lightweight account-linked preferences and metadata

The cloud account is not the primary execution environment for your workspace.

## What can sync

Examples of account-linked state include language preference, device registration, browser session state, and remote capability metadata.

## What remains local

Workspace files, installed runtimes, active host processes, and local WebUI state remain on the desktop host unless an explicit product flow moves them elsewhere.

## Device management

When troubleshooting account state, confirm which devices are currently registered and which account is active on each client. Most remote-access issues are easier to diagnose after verifying device binding first.
