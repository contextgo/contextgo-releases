## Local collaboration

Harness mode is the local collaboration model. Use it when the desktop host is the main workspace and you want multiple agent roles or coordinated task flows on that machine.

This mode is useful when local files, local tools, or installed runtimes are required for the task.

## Remote collaboration

Remote access extends the same host-based model. A browser or mobile client can connect to the desktop host, review status, and continue work without moving the workspace off the device.

Remote collaboration depends on the host remaining online and authenticated. If the host is unavailable, the remote client cannot continue the session independently.

## What to verify before starting

Before starting a collaborative workflow, confirm the following:

- The desktop host has the required files and runtimes.
- Any assistants used by the workflow are already configured.
- Remote access is enabled if browser or mobile clients will participate.

## Operational limits

Keep these boundaries in mind:

- Task execution still happens on the desktop host.
- Files remain in the host workspace unless you move them explicitly.
- Runtime failures on the host affect both local and remote sessions.
