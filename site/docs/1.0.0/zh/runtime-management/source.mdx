## 运行时状态

建议把运行时状态分为以下几类：

- 未安装
- 已安装
- 已认证或已配置
- 已就绪

已安装并不一定代表可以立即执行任务。有些运行时仍然需要完成 provider 登录或 API 配置。

## 安装与修复

运行时管理用于安装受支持的 CLI、重新检测状态，以及修复已损坏的安装路径。修复动作应只作用于当前运行时。

不要把修复动作当作整机清理工具。它不应覆盖无关的开发工具或全局配置。

## 就绪检查

正式开始任务前，建议至少确认：

- CLI 已被正确识别
- provider 登录已完成
- 所需 API Key 或配置可用
- 运行时能通过一次基础健康检查
