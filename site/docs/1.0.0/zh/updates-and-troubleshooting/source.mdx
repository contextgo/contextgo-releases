## 统一版本来源

公开发布统一来自 `contextgo/contextgo-releases`。官网下载页和桌面端内的更新入口都应指向同一份发布来源。

如果官网版本和应用内显示的更新版本不一致，应先按发布链路异常处理。

## 建议排查顺序

建议优先按影响范围最小的顺序检查：

1. 先确认账号登录和浏览器回跳是否正常。
2. 再确认目标设备是否已注册且在线。
3. 如问题出现在网页端或移动端，再检查远程访问状态。
4. 再检查运行时识别和 provider 认证。
5. 最后核对官网版本、应用内版本和发布仓库版本是否一致。

## 常见问题类型

这个页面适合作为以下问题的统一入口：

- 浏览器登录没有完成
- 设备绑定没有刷新
- 远程访问显示 unavailable
- 运行时已安装但无法执行
- 更新检查失败或版本异常

## 什么时候要核对版本

出现以下情况时，建议直接对比版本来源：

- 官网下载页显示一个版本，应用内更新显示另一个版本
- 用户反馈有更新提示，但安装包无法下载
- 发布仓库里已有版本，但官网还没有展示出来
