## 核心组成

ContextGo 由四个部分组成：

- 桌面主机：负责文件、工具、运行时和本地 WebUI。
- 云账号：负责登录、设备注册和账号关联状态。
- 浏览器/移动端：用于连接桌面主机的远程入口。
- 发布仓库：用于公开安装包、版本记录和下载产物。

## 各部分分别做什么

真正执行任务的是桌面端。无论是本地文件访问、运行时调用，还是依赖本机环境的连接器，都发生在桌面主机上。

云账号不负责替代桌面端执行，它主要负责把设备和账号关联起来，并让网页端或移动端能够找到这些设备。

浏览器端和移动端是访问入口，不是独立的云端运行环境。

## 版本来源

公开发布的安装包统一来自 `contextgo/contextgo-releases`。官网下载页和应用内更新应该引用同一份版本来源。

如果你发现官网版本和应用内更新版本不一致，应先按发布链路问题处理。

## 建议阅读顺序

建议按下面的顺序阅读文档：

1. 先读 `快速开始`，完成第一台设备配置。
2. 再读 `Features`，配置 Agent、Hooks、定时任务、运行时、Connectors 或技能。
3. 最后读 `Operations`，处理账号、远程访问、更新和排障。
