## 开始前请先确认

ContextGo 以桌面端作为主机。浏览器端和移动端只是连接入口，真正的工作区、文件和运行时仍然在桌面设备上。

开始安装前，先确认以下条件：

- 当前桌面设备允许安装应用。
- 可以正常打开浏览器完成登录。
- 当前网络没有拦截 ContextGo 的登录或更新请求。

## 1. 安装桌面端

从 `contextgo.io` 下载对应平台的桌面安装包并完成安装。官网的下载页和应用内更新都应读取 `contextgo/contextgo-releases` 中的发布产物。

首次启动时，应用会准备本地工作区、WebUI 运行环境，以及远程访问所需的设备侧服务。

## 2. 登录云账号

在桌面端中通过 GitHub 或 Google 登录。浏览器回跳完成后，当前设备会注册到你的 ContextGo 账号下。

这个账号主要用于身份识别、设备注册和轻量状态同步，不负责托管你的本地工作区或运行时。

## 3. 检查当前设备状态

登录完成后，确认以下项目正常：

- 当前设备出现在账号设备列表中。
- 本地 WebUI 可以正常打开。
- 设备状态显示为可用或在线。

如果后续要从浏览器或移动端访问，还要继续确认远程访问已经启用。

## 4. 按需测试远程访问

如果你的使用方式包含远程访问，请在桌面主机保持在线的情况下打开网页入口进行测试。只有桌面主机保持连接，远程页面才可用。

移动端访问也是同一套主机模型。上传、文件访问和运行时执行，最终仍然落在桌面设备上。

## 初始化失败时怎么检查

建议按这个顺序排查：

1. 确认桌面端能正常拉起浏览器登录页。
2. 确认浏览器登录完成后能正确回到 ContextGo。
3. 检查本机安全策略、终端防护或企业网络策略是否拦截辅助进程或网络请求。
4. 如当前设备环境受限较多，换一台普通网络环境的机器重新验证。
