## Connectors

Connectors 用于把外部上下文引入 ContextGo。来源可以是文件、文档、共享盘、聊天内容或结构化系统数据。

为每个 Connector 建议至少记录：

- 来源系统是什么
- 采用什么接入方式
- 需要哪些权限
- 接入后会向 Agent 或自动化暴露哪些内容

## 渠道

渠道用于把结果发送到目标位置，例如发布面、通知路径或交付系统。一条工作流可以在本地处理完内容后，再通过一个或多个渠道输出。

## 典型链路

常见链路如下：

1. 接入来源系统。
2. 在 Agent、Hook 或定时任务中使用这些上下文。
3. 通过配置好的渠道输出结果。

## 权限建议

只授予当前工作流必需的最小权限。如果某个 Connector 或渠道需要高权限访问，启用前应先完成权限评估。
