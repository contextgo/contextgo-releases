## 本地协作

Harness 模式用于桌面主机上的本地协作。适合需要多个 Agent 角色、但又依赖本机文件、工具或运行时的工作流。

如果主要工作内容都在当前桌面设备上，这通常是最直接的协作方式。

## 远程协作

远程访问是在同一套主机模型上增加浏览器端或移动端入口。它可以查看状态、继续任务或参与控制，但不会把工作区从桌面端迁走。

如果桌面主机离线或失去连接，远程客户端也无法单独继续执行。

## 开始协作前的检查项

开始协作前，建议先确认：

- 桌面主机已经具备所需文件和运行时。
- 本次使用的 Assistant 已经配置完成。
- 如果需要远程参与，远程访问功能已经启用。

## 使用边界

使用协作模式时，默认遵循以下边界：

- 真正执行任务的是桌面主机。
- 文件默认留在主机工作区内。
- 主机运行时异常会同时影响本地和远程会话。
